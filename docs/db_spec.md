# Theseus Insight - Database Specification v3.0

> PostgreSQL schema (with pgvector) created automatically during application start-up

---

## 1. Purpose & Scope

The database stores **all persistent artefacts** generated by the Theseus Insight pipeline:

* research **papers** that have been downloaded, scored and embedded
* **full-text content** for papers  
* generated **newsletters** and **podcasts**
* **research agent** runs and state
* **literature reviews** and **mind map reports**
* **topic evolution** and **trends analysis**
* **research interests** tracking and metrics
* operational **logs**
* user-editable **settings** (key/value)
* **model catalog** and **model providers**
* **label summaries** cache

Database access is configured via the `DATABASE_URL` environment variable.

---

## 2. Physical Characteristics

| Property             | Default value                                                                                                                                                    | Notes                                                  |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------ |
| **Engine**           | PostgreSQL 14+ with pgvector extension |
|                          | Hosted database (local or cloud) |
| **Connection string**| `postgresql://postgres:postgres@localhost:5432/theseus` |
|                          | Provided via `DATABASE_URL` environment variable |
| **Connection model** | Uses psycopg2 connection pooling |
| **Vector extension** | pgvector for semantic similarity search |
| **Full-text search** | PostgreSQL native tsvector/tsquery |

---

## 3. Entity-Relationship Overview

```
model_providers ──< model_catalog
                    
papers ──< paper_fulltext
       ──< paper_topics ──> topics ──< topic_metrics
       ──< paper_research_interests ──> research_interests ──< research_interest_metrics
       ──< mindmap_reports

research_runs ──< research_agent_state
              ──< lit_reviews

newsletters, podcasts, settings, logs, tasks (independent)
label_summaries (cache)
```

Key relationships:
- `paper_fulltext.paper_id → papers.id` (ON DELETE CASCADE)
- `mindmap_reports.seed_paper_id → papers.id` (ON DELETE CASCADE)
- `research_agent_state.task_id → research_runs.task_id` (ON DELETE CASCADE)
- Junction tables for many-to-many relationships

## 5. PostgreSQL-Specific Features

### 5.1 Vector Search with pgvector

The `embedding` columns use pgvector's `VECTOR(768)` type for efficient semantic similarity search:

```sql
-- Find similar papers using cosine distance
SELECT *, 1 - (embedding <=> %s) AS similarity_score
FROM papers 
WHERE embedding IS NOT NULL 
  AND (1 - (embedding <=> %s)) >= 0.7
ORDER BY embedding <=> %s
LIMIT 10;
```

### 5.2 Full-Text Search

PostgreSQL's native full-text search using generated tsvector columns:

```sql
-- Search papers by content
SELECT * FROM papers 
WHERE fts @@ plainto_tsquery('english', 'machine learning')
ORDER BY ts_rank(fts, plainto_tsquery('english', 'machine learning')) DESC;

-- Search full-text content
SELECT p.*, pf.content 
FROM papers p
JOIN paper_fulltext pf ON p.id = pf.paper_id
WHERE pf.fts @@ plainto_tsquery('english', 'neural networks');
```

### 5.3 Hybrid Search

Combines semantic and keyword search across multiple tables:

```sql
SELECT p.*, 
  (1 - (p.embedding <=> %s)) AS semantic_score,
  ts_rank(p.fts, plainto_tsquery('english', %s)) AS keyword_score,
  ((1 - (p.embedding <=> %s)) * 0.6 + ts_rank(p.fts, plainto_tsquery('english', %s)) * 0.4) AS hybrid_score
FROM papers p
WHERE p.embedding IS NOT NULL
ORDER BY hybrid_score DESC;
```

### 5.4 Topic Analysis

Complex queries for topic evolution and trends:

```sql
-- Get trending topics with growth metrics
SELECT t.label, tm.doc_count, tm.growth_rate, tm.forecast_3m
FROM topics t
JOIN topic_metrics tm ON t.id = tm.topic_id
WHERE tm.period_type = 'month'
  AND tm.period_start >= CURRENT_DATE - INTERVAL '3 months'
ORDER BY tm.growth_rate DESC;
```

## 6. Performance & Indexing Notes

### 6.1 Indexes

Indexes automatically created by the schema:

```sql
-- Primary keys and unique constraints
-- Foreign key indexes
-- Vector search indexes
CREATE INDEX IF NOT EXISTS idx_papers_fts ON papers USING GIN (fts);
CREATE INDEX IF NOT EXISTS idx_paper_fulltext_fts ON paper_fulltext USING GIN (fts);

-- Performance indexes
CREATE INDEX IF NOT EXISTS idx_papers_score ON papers (score DESC);
CREATE INDEX IF NOT EXISTS idx_tasks_status ON tasks (status);
CREATE INDEX IF NOT EXISTS idx_tasks_start_time ON tasks (start_time DESC);
CREATE INDEX IF NOT EXISTS idx_research_runs_status ON research_runs (status);
CREATE INDEX IF NOT EXISTS idx_research_runs_created_at ON research_runs (created_at DESC);

-- Topic and trends indexes
CREATE INDEX IF NOT EXISTS idx_topics_label ON topics (label);
CREATE INDEX IF NOT EXISTS idx_topic_metrics_topic_id ON topic_metrics (topic_id);
CREATE INDEX IF NOT EXISTS idx_topic_metrics_period ON topic_metrics (period_start DESC);
CREATE INDEX IF NOT EXISTS idx_paper_topics_paper_id ON paper_topics (paper_id);
CREATE INDEX IF NOT EXISTS idx_paper_topics_topic_id ON paper_topics (topic_id);

-- Research interest indexes
CREATE INDEX IF NOT EXISTS idx_paper_research_interests_paper_id ON paper_research_interests (paper_id);
CREATE INDEX IF NOT EXISTS idx_paper_research_interests_ri_id ON paper_research_interests (research_interest_id);

-- Model catalog indexes
CREATE INDEX IF NOT EXISTS idx_model_catalog_alias ON model_catalog (alias);
CREATE INDEX IF NOT EXISTS idx_model_catalog_provider ON model_catalog (provider_name);
CREATE INDEX IF NOT EXISTS idx_model_catalog_type ON model_catalog (model_type);
```

### 6.2 Vector Index Tuning

For optimal vector search performance on large datasets:

```sql
-- Adjust lists parameter based on data size
-- Rule of thumb: lists = rows / 1000
CREATE INDEX idx_papers_embedding ON papers 
USING ivfflat (embedding vector_cosine_ops) 
WITH (lists = 100);

CREATE INDEX idx_paper_fulltext_embedding ON paper_fulltext 
USING ivfflat (embedding vector_cosine_ops) 
WITH (lists = 50);
```

### 6.3 Connection Pooling

The application uses connection pooling for optimal performance:
- Connection pool size: configured per environment
- Idle timeout: prevents connection leaks
- Max connections: prevents database overload

## 8. Common Query Patterns

### 8.1 Research Agent Queries

```sql
-- Get active research runs
SELECT task_id, research_question, status, created_at
FROM research_runs 
WHERE status = 'processing'
ORDER BY created_at DESC;

-- Get research run with full state
SELECT rr.*, array_agg(ras.state_json ORDER BY ras.timestamp) AS states
FROM research_runs rr
LEFT JOIN research_agent_state ras ON rr.task_id = ras.task_id
WHERE rr.task_id = %s
GROUP BY rr.id;
```

### 8.2 Topic Evolution Queries

```sql
-- Topic growth over time
SELECT t.label, 
       tm.period_start,
       tm.doc_count,
       tm.growth_rate
FROM topics t
JOIN topic_metrics tm ON t.id = tm.topic_id
WHERE t.label = %s
  AND tm.period_type = 'month'
ORDER BY tm.period_start DESC;
```

### 8.3 Mind Map Queries

```sql
-- Recent mind map reports
SELECT id, title, seed_paper_title, created_at
FROM mindmap_reports
ORDER BY created_at DESC
LIMIT 10;
```
