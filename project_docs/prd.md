# Mind‑Map Explorer – Product Requirements Document

---

## 1  Background & Problem Statement

Researchers often discover a single relevant paper but struggle to **visualise its intellectual neighbourhood**—related works, earlier inspirations, and follow‑up studies. Existing discovery tools provide linear lists, not an **interactive concept map** that can be expanded, filtered, annotated, **and optionally enriched with full‑text insights**.

The organisation maintains a **SQLite database (≈46 k records)** with title, abstract and vector embeddings **and surfaces them today through an internal *********************************************************************************************************************************************************************************************************************PapersHistory********************************************************************************************************************************************************************************************************************* library**. The new Mind‑Map Explorer will plug directly into this library: when a user views any entry in PapersHistory they can click “Open Mind Map” to launch the visual exploration view in a side panel or separate tab. (≈46 k records)\*\* with title, abstract and vector embeddings. We want to leverage this data **plus on‑demand PDF parsing** and interchangeable LLMs to generate an explorable mind map around any seed paper.

---

## 2  Goals

1. **Generate** a mind map of the *k* most relevant papers to a chosen seed, enriched with 1‑sentence summaries generated by a pluggable LLM.
2. **Provide on‑demand PDF parsing** (max 20 papers per request) so users can dive into full‑text when needed without pre‑processing the entire corpus.
3. **Display** the map in a React application with pan/zoom, node‑click details, iterative expansion and PDF‑aware highlights.
4. **Run offline‑first** using a local LLM runtime and `sqlite-vec`; optionally call remote LLM APIs when configured.
5. **Persist** user layouts & notes for later sessions.
6. **Integrate** seamlessly with the existing **PapersHistory** library so that any paper record can launch this mind‑map view on demand.

---

## 5  Functional Requirements

### 5.1 Paper Selection

* **FR‑1.1** Autocomplete search over titles & authors (FTS5).
* **FR‑1.2** Accept DOI/URL paste to resolve paper ID.
* **FR‑1.3** Provide a **`useMindMap()`**\*\* hook\*\* (or equivalent) exported by PapersHistory that allows callers to open the Mind‑Map Explorer for the currently selected paper.

### 5.2 Mind‑Map Generation Mind‑Map Generation

* **FR‑2.1** Retrieve top‑*k* neighbours via cosine similarity using `sqlite-vec`.
* **FR‑2.2** Summarise each neighbour via **LLMFactory** – interchangeable runtime that can route to local or remote LLM based on config.
* **FR‑2.3** Build a hierarchical mind‑map data structure ordered by similarity distance.

### 5.3 On‑Demand PDF Parsing  *(NEW)*

* **FR‑3.1** Allow user to select up to **20 papers per request** for full‑text fetch and parse.
* **FR‑3.2** For each PDF, extract: Introduction, Methods, Results, Conclusion + figures/tables references.
* **FR‑3.3** Store parsed text and derived embeddings in a secondary table **`paper_fulltext`** keyed by `paper_id`.
* **FR‑3.4** Parsing pipeline uses PyMuPDF; fallback to OCR (Tesseract) when text layer missing.
* **FR‑3.5** Display parse progress in UI and merge new insights into the existing mind map (e.g., highlight if methods overlap).

### 5.4 Interactive Canvas

* **FR‑4.1** Pan/zoom via mouse or trackpad.
* **FR‑4.2** Hover → tooltip; click → side drawer.
* **FR‑4.3** Right‑click → *“Expand node”* (runs backend expansion with new seed).
* **FR‑4.4** Node drag rearranges layout; positions persist per user in IndexedDB.

### 5.5 Filtering & Search

* **FR‑5.1** Dynamic filters (year ≥ X, venue ∈ Y, similarity ≤ δ).
* **FR‑5.2** Keyword search highlights matching nodes (uses FTS rank).

### 5.6 Export & Share

* **FR‑6.1** Export current view to PNG/SVG.

* **FR‑6.2** Save/Load map JSON snapshots.

## 8  Algorithms & Processing

| Step                                                                                                                                                          | Details |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------- |
| **P‑0** *(optional)* If user requests PDF parsing, download & extract up to 20 PDFs → populate `paper_fulltext`, compute embeddings using `EmbeddingService`. |         |
| **E‑1** Embed seed abstract (skip if cached).                                                                                                                 |         |
| **E‑2** Similarity search via `sqlite-vec` (cosine distance) → top‑*k* records.                                                                               |         |
| **E‑3** Parallel summarisation with `LLMFactory`. If local runtime returns low confidence, factory routes call to remote API.                                 |         |
| **E‑4** Optional MMR re‑ranking.                                                                                                                              |         |
| **E‑5** Build graph JSON and stream to UI.                                                                                                                    |         |

## 9  System Architecture

```text
[React App] ⇄ /api/ws (WebSocket) ⇄ [FastAPI Backend]
                                        │
                                        ├─ LangGraph Runner
                                        │   └─ Nodes: SelectSeed › EmbedSeed › Retrieve › Summarise › BuildMap
                                        │
                                        ├─ PDF Parser Service (PyMuPDF + OCR)
                                        │
                                        └─ SQLite  ← sqlite‑vec, FTS5, paper_fulltext
```

* **LLMFactory** abstracts away model choice; plugins: *local‑ggml*, *openai‑api*, *ollama*, …
* **State streaming**: LangGraph snapshots → WebSocket patches → React reducer.

## 10  API Contracts

### 10.1 `POST /expand`

| Field     | Type   | Description                  |
| --------- | ------ | ---------------------------- |
| `paperId` | string | Seed paper id                |
| `k`       | int    | Neighbour count (default 15) |
| `filters` | object | `{yearMin, venue, keyword}`  |
| `llmMode` | enum   | `local \| hybrid`            |

### 10.2 `POST /parse-pdfs`

Batch‑parse up to **20** papers’ PDFs.

| Field      | Type  | Description              |
| ---------- | ----- | ------------------------ |
| `paperIds` | array | List of paper ids (≤ 20) |

*Responds* with incremental progress events (`paperId`, `status`, `error?`).

### 10.3 `GET /paper/:id`

Return metadata, abstract, cached summary, and flags indicating if full‑text is available.

## 13  Risks & Mitigations

| Risk                                       | Impact            | Likelihood | Mitigation                                      |
| ------------------------------------------ | ----------------- | ---------- | ----------------------------------------------- |
| PDF parsing rate limits on publisher sites | Missing full‑text | Medium     | Respect robots.txt, user supplies PDFs manually |
| `sqlite-vec` performance on >50 k rows     | Slow retrieval    | Medium     | Build ANN index; pagination                     |
| LLM summary quality                        | Low readability   | Medium     | Confidence routing; allow user edits            |
| Canvas clutter                             | Poor UX           | Medium     | Node cap, automatic clustering                  |
