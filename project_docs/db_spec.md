# Theseus Insight - Database Specification v0.9.0

> SQLite schema created automatically by `PaperDatabase._initialize_db()` during application start-up

---

## 1. Purpose & Scope

The database stores **all persistent artefacts** generated by the Phase 1 pipeline:

* research **papers** that have been downloaded, scored and embedded
* generated **newsletters** and **podcasts**
* operational **logs**
* use-editable **settings** (key/value)
* configured **model providers** and individual **models**

The default file path is `data/papers.db`; it can be overridden via the CLI flag `--data-path` or the env-var `THESEUS_DB_PATH`.

---

## 2. Physical Characteristics

| Property             | Default value                                                                                                                                                    | Notes                                                  |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------ |
| **Engine**           | SQLite 3                                                                                                                                                         | Zero-config for local dev; no external server          |
| **File location**    | `data/papers.db`                                                                                                                                                 | Created on first run; parent directory is auto-created |
| **Connection model** | One short-lived connection per CRUD call (context-manager wraps `sqlite3.connect`)                                                                               |                                                        |
| **Future migration** | Schema is ANSI-SQL compliant; a `postgres.sql` migration script can be generated loss-lessly (all TEXT → `text`, REAL → `double precision`, BOOLEAN → `boolean`) |                                                        |

---

## 3. Entity-Relationship Overview

```
model_providers 1 ────< models
                         ▲
papers ───────┐
newsletters   ├── stored artefacts
podcasts ─────┘
settings  (key/value)
logs      (append-only)
```

Only **one foreign-key** exists today: `models.provider_id → model_providers.id` (ON DELETE CASCADE recommended when ported to Postgres).

## 4. Table Specifications

### 4.1 `papers`

| Column              | Type    | Constraints           | Description                |
| ------------------- | ------- | --------------------- | -------------------------- |
| `id`                | INTEGER | **PK** AUTOINCREMENT  | Surrogate key              |
| `title`             | TEXT    | NOT NULL              | Paper title                |
| `abstract`          | TEXT    | NOT NULL              | Abstract text              |
| `date`              | TEXT    | NOT NULL `YYYY-MM-DD` | Publication date           |
| `date_run`          | TEXT    | NOT NULL `YYYY-MM-DD` | Processing date            |
| `score`             | REAL    | NOT NULL (0-10)       | Relevance score            |
| `rationale`         | TEXT    | NOT NULL              | LLM explanation            |
| `related`           | BOOLEAN | NOT NULL              | `TRUE` if on-topic         |
| `cosine_similarity` | REAL    | NOT NULL              | Embedding similarity       |
| `url`               | TEXT    | NOT NULL              | PDF/landing-page URL       |
| `embedding_model`   | TEXT    | NOT NULL              | Name/version of model used |

### 4.2 `logs`

| Column         | Type    | Constraints          | Description                        |
| -------------- | ------- | -------------------- | ---------------------------------- |
| `id`           | INTEGER | **PK** AUTOINCREMENT |                                    |
| `status_code`  | INTEGER | NOT NULL             | HTTP-like code or custom error key |
| `status`       | TEXT    | NOT NULL             | Human-readable message             |
| `datetime_run` | TEXT    | `YYYY-MM-DD hh:mm`   | Optional timestamp                 |

### 4.3 `newsletters`

| Column       | Type    | Constraints          | Description               |
| ------------ | ------- | -------------------- | ------------------------- |
| `id`         | INTEGER | **PK** AUTOINCREMENT |                           |
| `content`    | TEXT    | NOT NULL             | Markdown newsletter body  |
| `start_date` | TEXT    | NOT NULL             | First paper date included |
| `end_date`   | TEXT    | NOT NULL             | Last paper date included  |
| `date_sent`  | TEXT    | NOT NULL             | Email dispatch date       |

### 4.4 `podcasts`

| Column        | Type    | Constraints          | Description                   |
| ------------- | ------- | -------------------- | ----------------------------- |
| `id`          | INTEGER | **PK** AUTOINCREMENT |                               |
| `title`       | TEXT    | NOT NULL             | Episode title                 |
| `date`        | TEXT    | NOT NULL             | Generation date               |
| `script`      | TEXT    | NOT NULL             | JSON string of dialogue items |
| `description` | TEXT    | NOT NULL             | Show-notes                    |

### 4.5 `settings`

Key-value store for small configuration items.

| Column  | Type | Constraints | Description                   |
| ------- | ---- | ----------- | ----------------------------- |
| `key`   | TEXT | **PK**      | Unique setting name           |
| `value` | TEXT | NOT NULL    | Serialized JSON or plain text |

Typical keys: `email_recipients`, `visualizer_defaults`, `api_keys`, etc.

### 4.6 `model_providers`

| Column | Type    | Constraints          | Description                                           |
| ------ | ------- | -------------------- | ----------------------------------------------------- |
| `id`   | INTEGER | **PK** AUTOINCREMENT |                                                       |
| `name` | TEXT    | NOT NULL UNIQUE      | Provider label - e.g. `openai`, `anthropic`, `ollama` |

### 4.7 `models`

| Column         | Type                    | Constraints                             | Description                           |
| -------------- | ----------------------- | --------------------------------------- | ------------------------------------- |
| `id`           | INTEGER                 | **PK** AUTOINCREMENT                    |                                       |
| `provider_id`  | INTEGER                 | NOT NULL, **FK** → `model_providers.id` |                                       |
| `name`         | TEXT                    | NOT NULL                                | Model identifier (e.g. `gpt-4o-mini`) |
| `config_json`  | TEXT                    | nullable                                | Provider-specific JSON blob           |
| **Unique key** | (`provider_id`, `name`) | Prevents duplicates                     |                                       |

## 5. Expected Workflows

### Insert examples

```sql
INSERT INTO papers (title,abstract,date,date_run,score,rationale,related,
                    cosine_similarity,url,embedding_model)
VALUES (...);

INSERT INTO settings (key,value)
VALUES ('email_recipients', '["user@example.com"]')
ON CONFLICT(key) DO UPDATE SET value=excluded.value;
```

### Query examples

**Latest 10 ranked papers (score ≥ 8)**

```sql
SELECT title, score, date
FROM papers
WHERE score >= 8
ORDER BY date_run DESC
LIMIT 10;
```

**All models for a provider**

```sql
SELECT m.name, m.config_json
FROM models m
JOIN model_providers p ON p.id = m.provider_id
WHERE p.name = 'openai';
```

## 6. Performance & Indexing Notes

* **Primary keys** implicitly create clustered indexes in SQLite.
* The composite `UNIQUE(provider_id, name)` on `models` also yields an index.
* If query latency becomes an issue once paper counts exceed \~100 k, add:

```sql
CREATE INDEX idx_papers_date ON papers(date);
CREATE INDEX idx_papers_score ON papers(score DESC);
```
